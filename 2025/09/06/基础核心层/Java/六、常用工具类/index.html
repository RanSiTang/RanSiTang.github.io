六、常用工具类

1. Math

1.1 简介 (Introduction)

java.lang.Math 类是一个工具类，提供了执行基本数学运算所需的方法，如初等指数、对数、平方根和三角函数。它包含了大量的静态方法，方便我们随时调用。

1.2 核心特点

  • final 类: Math 类被声明为 final，因此不能被继承。
  • 私有构造函数: 它的构造函数是 private 的，这意味着你无法创建 Math 类的实例。这强制你只能通过类名直接调用其静态方法（例如 Math.abs(-10)）。
  • 静态成员: 所有的字段（如 PI, E）和方法都是 static 的。
  • 无需导入: 因为 Math 类位于 java.lang 包下，所以在任何Java代码中都可以直接使用，无需显式 import。

1.3 常用方法详解与示例

1.3.1 基本运算

方法签名 描述
static double abs(double a) 返回一个数的绝对值（有 int, long, float 的重载版本）。
static double pow(double a, double b) 返回第一个参数的第二个参数次幂的值，即 a^b。
static double sqrt(double a) 返回一个数的平方根。
static double cbrt(double a) 返回一个数的立方根。
static double max(double a, double b) 返回两个数中的较大值（有 int, long, float 的重载版本）。
static double min(double a, double b) 返回两个数中的较小值（有 int, long, float 的重载版本）。

1.3.2 取整运算

方法签名 描述
static double ceil(double a) 向上取整。返回大于或等于参数的最小整数（类型为double）。
static double floor(double a) 向下取整。返回小于或等于参数的最大整数（类型为double）。
static long round(double a) 四舍五入。返回最接近参数的 long 型整数。

注意: Math.round(1.5) 结果是2，Math.round(-1.5) 结果是-1。其算法是 Math.floor(x+0.5)。

1.3.3 随机数

方法签名 描述
static double random() 返回一个 [0.0, 1.0) 范围内的伪随机 double 值。

高级技巧:

  • 生成 [min, max] 范围的随机整数: (int) (Math.random() * (max - min + 1) + min)
  • 在并发或需要更高质量随机数的场景下，推荐使用 java.util.Random 类或 java.util.concurrent.ThreadLocalRandom。

1.3.4 三角函数与常数

成员 描述
static final double PI 圆周率 π。
static final double E 自然对数的底数 e。
static double sin(double a) 返回角的三角正弦（参数为弧度）。
static double cos(double a) 返回角的三角余弦（参数为弧度）。
static double toRadians(double angdeg) 将角度转换为弧度。
static double toDegrees(double angrad) 将弧度转换为角度。

2. System

2.1 简介 (Introduction)

java.lang.System 类同样是一个工具类，它提供了对标准输入、标准输出、错误输出流的访问，以及访问外部定义的属性和环境变量、加载文件和库、以及执行数组复制等系统级操作。

2.2 核心特点

  • final 类: 不可被继承。
  • 私有构造函数: 不可被实例化。
  • 静态成员: 所有成员和方法都是静态的。
  • 无需导入: 位于 java.lang 包下。

2.3 常用方法详解与示例

2.3.1 标准I/O流

字段 类型 描述
static InputStream in java.io.InputStream 标准输入流。通常对应键盘输入。
static PrintStream out java.io.PrintStream 标准输出流。通常对应控制台输出。System.out.println() 就是用的它。
static PrintStream err java.io.PrintStream 标准错误流。通常也对应控制台，但用于输出错误信息。

2.3.2 系统操作

方法签名 描述
static void exit(int status) 终止当前运行的 Java 虚拟机。status 为 0 表示正常退出，非 0 表示异常退出。谨慎使用，会导致程序立即中断。
static void gc() 建议 JVM 执行垃圾回收。这只是一个请求，JVM 不一定会立即执行。
static long currentTimeMillis() 返回自 1970 年 1 月 1 日 00:00:00 UTC 以来的毫秒数。常用于计算代码执行时间。
static long nanoTime() 返回 JVM 高精度时间源的当前值，单位是纳秒。主要用于精确测量时间间隔，不适合用于获取当前日期时间。
static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length) 高效地从源数组复制数据到目标数组。这是一个 native 方法，性能远高于循环复制。
static String getProperty(String key) 获取指定键的系统属性，如 java.version, os.name 等。

3. Objects

3.1 简介 (Introduction)

java.util.Objects 是一个在 JDK 7 中引入的工具类，它包含了一系列静态实用方法，用于操作对象或在操作前检查某些条件。这个类的主要设计目标是简化代码和防止 NullPointerException。

3.2 核心特点

  • 需要导入: 位于 java.util 包，使用前需要 import java.util.Objects;。
  • 空指针安全 (Null-safe): 大部分方法都对 null 进行了处理，从而避免了 NullPointerException。
  • 静态方法: 所有方法都是 static 的。
  • 私有构造函数: 不可被实例化。

3.3 常用方法详解与示例

3.3.1 空指针检查

方法签名 描述
static <T> T requireNonNull(T obj) 如果 obj 不为 null，则返回 obj 本身；否则，抛出 NullPointerException。
static <T> T requireNonNull(T obj, String message) 与上一个方法类似，但在抛出异常时会使用指定的 message。
static boolean isNull(Object obj) 判断对象是否为 null。等价于 obj == null，但更具可读性，常用于 Stream API。
static boolean nonNull(Object obj) 判断对象是否不为 null。等价于 obj != null，同样常用于 Stream API。

最佳实践: 在构造方法和普通方法的参数校验中，使用 requireNonNull 来实现“快速失败（Fail-fast）”，能让问题尽早暴露。

3.3.2 对象比较与哈希

方法签名 描述
static boolean equals(Object a, Object b) 比较两个对象是否相等，内部处理了 null 的情况。如果 a 和 b 都为 null，返回 true。如果只有一个为 null，返回 false。否则，调用 a.equals(b)。
static int hashCode(Object o) 如果 o 不为 null，返回 o.hashCode()；如果 o 为 null，返回 0。
static int hash(Object... values) 为一组对象生成一个哈希码。常用于在 hashCode() 方法中组合多个字段的哈希值。

最佳实践: 在重写 equals() 方法时，推荐使用 Objects.equals()。在重写 hashCode() 方法时，推荐使用 Objects.hash()。

3.3.3 其他实用方法

方法签名 描述
static String toString(Object o) 如果 o 不为 null，返回 o.toString()；如果 o 为 null，返回字符串 “null”。
static String toString(Object o, String nullDefault) 如果 o 不为 null，返回 o.toString()；如果 o 为 null，返回 nullDefault。

4. BigInteger

4.1 简介 (Introduction)

java.math.BigInteger 是一个用于表示任意精度整数的类。当一个整数的大小超过了 long 类型的最大值（Long.MAX_VALUE，约 9 x 10^18）时，就必须使用 BigInteger。它内部用一个 int[] 数组来存储这个超大的整数，因此理论上它可以表示的整数大小只受限于可用内存。

4.2 核心特点

  • 不可变性 (Immutability): BigInteger 对象是不可变的。任何算术运算（如加、减、乘、除）都会返回一个新的 BigInteger 对象，而不会修改原始对象。
  • 任意精度: 可以表示比 long 更大的整数。
  • 面向对象: 算术运算通过方法调用完成，而不是使用 +, -, *, / 等操作符。

4.3 构造方法

构造方法 描述
BigInteger(String val) 最常用。将一个表示十进制整数的字符串转换为 BigInteger 对象。
BigInteger(String val, int radix) 将指定进制（radix）的字符串转换为 BigInteger。
static BigInteger valueOf(long val) 将 long 类型的值转换为 BigInteger，有性能优化。

4.4 常用方法详解与示例

方法签名 描述
BigInteger add(BigInteger val) 加法。
BigInteger subtract(BigInteger val) 减法。
BigInteger multiply(BigInteger val) 乘法。
BigInteger divide(BigInteger val) 除法（取整）。
BigInteger remainder(BigInteger val) 取余。
BigInteger[] divideAndRemainder(BigInteger val) 返回一个包含商和余数的 BigInteger 数组。
int compareTo(BigInteger val) 比较大小。返回 -1 (小于), 0 (等于), 1 (大于)。
BigInteger pow(int exponent) 计算当前 BigInteger 的 exponent 次幂。
BigInteger abs() 返回绝对值。
int intValue(), long longValue() 转换为 int 或 long。注意：如果 BigInteger 的值超出范围，会发生数据丢失！

5. BigDecima

5.1 简介 (Introduction)

java.math.BigDecimal 是用于表示任意精度带符号十进制数的类。它解决了 float 和 double 类型在进行浮点数运算时存在的精度丢失问题。因此，在需要精确计算的场景，尤其是金融领域（如货币计算），必须使用 BigDecimal。

经典面试题：为什么 0.1 + 0.2 不等于 0.3？
因为 float 和 double 是基于二进制的浮点表示法，无法精确地表示像 0.1 这样的十进制小数，只能无限接近。而 BigDecimal 内部使用一个 BigInteger (unscaled value) 和一个 int (scale) 来表示数值，是基于十进制的，从而保证了精度。

5.2 核心特点

  • 不可变性 (Immutability): 与 BigInteger 相同，运算会返回新对象。
  • 精确十进制表示: 避免了二进制浮点数的舍入误差。
  • 需要指定舍入模式: 在进行除法等可能产生无限小数的运算时，必须显式指定舍入模式 (RoundingMode)，否则可能抛出 ArithmeticException。

5.3 构造方法 (陷阱与最佳实践)

构造方法 描述
BigDecimal(String val) 强烈推荐。将字符串精确转换为 BigDecimal。
static BigDecimal valueOf(double val) 推荐。内部使用 Double.toString(val)，避免了 new BigDecimal(double) 的精度问题。
BigDecimal(double val) 严禁使用！此构造方法会引入 double 本身的精度误差。例如 new BigDecimal(0.1) 的实际值并非 0.1。

5.4. 常用方法详解与示例

方法签名 描述
BigDecimal add(BigDecimal augend) 加法。
BigDecimal subtract(BigDecimal subtrahend) 减法。
BigDecimal multiply(BigDecimal multiplicand) 乘法。
BigDecimal divide(BigDecimal divisor, int scale, RoundingMode roundingMode) 最常用的除法。scale 指定小数点后保留的位数，roundingMode 指定舍入模式。
int compareTo(BigDecimal val) 推荐用于比较值的大小。返回 -1, 0, 1。它只比较数值，忽略 scale。
boolean equals(Object x) 比较对象是否相等。注意：它会同时比较值和 scale。例如 new BigDecimal("2.0") 和 new BigDecimal("2.00") 不相等。
BigDecimal setScale(int newScale, RoundingMode roundingMode) 设置小数位数和舍入模式，返回一个新对象。

常用舍入模式 RoundingMode

模式 描述
RoundingMode.HALF_UP 四舍五入。
RoundingMode.DOWN 直接截断，不进行任何舍入。
RoundingMode.UP 远离零方向舍入。
RoundingMode.CEILING 向正无穷方向舍入。
RoundingMode.FLOOR 向负无穷方向舍入。

6. 正则表达式

6.1 简介 (Introduction)

正则表达式（Regex）是一种强大的字符串处理工具，它定义了一个搜索模式，用于匹配、查找、替换和分割字符串。Java 通过 java.util.regex 包提供了完整的正则表达式支持。

6.2 核心类

  • Pattern: 正则表达式的编译表示。一个正则表达式字符串首先被编译成一个 Pattern 对象，这个过程有性能开销，因此对于需要重复使用的正则表达式，应将其编译一次并重用。
  • Matcher: 模式匹配器。一个 Matcher 对象是 Pattern 对象在一个具体输入字符串上的匹配状态机。它提供了多种查找和匹配方法。

6.3 常用元字符 (Metacharacters)

类别 元字符 描述
字符类 . 匹配除换行符外的任意单个字符。
\d 匹配一个数字，等价于 [0-9]。
\D 匹配一个非数字。
\w 匹配一个单词字符（字母、数字、下划线），等价于 [a-zA-Z0-9_]。
\W 匹配一个非单词字符。
\s 匹配一个空白字符（空格、制表符、换行符等）。
[...] 匹配方括号中的任意一个字符。例如 [abc] 匹配 ‘a’ 或 ‘b’ 或 ‘c’。
[^...] 匹配不在方括号中的任意一个字符。
量词 * 匹配前面的元素零次或多次。
+ 匹配前面的元素一次或多次。
? 匹配前面的元素零次或一次。
{n} 匹配前面的元素恰好 n 次。
{n,} 匹配前面的元素至少 n 次。
{n,m} 匹配前面的元素至少 n 次，至多 m 次。
边界 ^ 匹配输入的开头。
$ 匹配输入的结尾。
\b 匹配单词边界。
分组 (...) 将多个字符作为一个单元进行分组，并捕获匹配的内容。

6.4 Java 中的使用方式

6.4.1 String 类的便捷方法

String 类内置了一些使用正则表达式的便捷方法，适合简单的匹配场景。

  • boolean matches(String regex): 判断整个字符串是否完全匹配给定的正则表达式。
  • String replaceAll(String regex, String replacement): 替换所有匹配的子串。
  • String[] split(String regex): 根据正则表达式分割字符串。

6.4.2 Pattern 和 Matcher 的标准流程 (更强大灵活)

  1. 编译模式: Pattern pattern = Pattern.compile(regex);
  2. 创建匹配器: Matcher matcher = pattern.matcher(inputString);
  3. 执行匹配:
    • matcher.matches(): 尝试将整个输入序列与该模式匹配（类似 String.matches）。
    • matcher.find(): 扫描输入序列，查找与该模式匹配的下一个子序列。常与 while 循环一起使用，找出所有匹配项。
    • matcher.group(int group): 在 find() 或 matches() 成功后，返回由给定捕获组捕获的子序列。group(0) 代表整个匹配。

7. 时间类

7.1 JDK 7及之前的时间API【过时】

7.1.1 Date类

1. 概述
  • 简介: Date 类是Java早期用于表示特定时间点的类。其内部本质上是一个包装了 long 类型毫秒值的对象，该毫秒值表示自UTC 1970年1月1日00:00:00以来的时间戳。
  • 核心问题: 除了 getTime() 和 setTime() 等少数方法外，大部分用于日期和时间分量计算的方法（如 getYear(), getMonth()）都已被废弃，因为它们的设计存在严重缺陷。Date 对象本身是可变的。
2. 常用方法
方法签名 描述
Date() 创建一个表示当前时间的 Date 对象。
Date(long date) 根据指定的毫秒值创建 Date 对象。
long getTime() 返回自纪元以来的毫秒数。
void setTime(long time) 设置 Date 对象的时间（修改自身）。

7.1.2 Calender

1. 概述
  • 简介: Calendar 是一个抽象类，用于弥补 Date 类在日期时间计算和操作上的不足。它提供了设置和获取年、月、日、时、分、秒等字段的方法。通常通过 Calendar.getInstance() 获取其子类 GregorianCalendar 的实例。
  • 核心问题: 对象可变、月份从0开始、API冗长、时区处理依然不便。
2. 常用方法
方法签名 描述

int get(int field) 获取指定字段的值，如 Calendar.YEAR。
void set(int field, int value) 设置指定字段的值。
void add(int field, int amount) 对指定字段进行加减运算。
Date getTime() 将 Calendar 对象转换回 Date 对象。
void setTime(Date date) 使用 Date 对象设置 Calendar 的时间。

7.1.2 SimpleDateFormat

1. 概述
  • 简介: SimpleDateFormat 是一个用于以特定格式（如 “yyyy-MM-dd HH:mm:ss”）格式化（日期 -> 文本）和解析（文本 -> 日期）日期的具体类。
  • 核心问题: 非线程安全。在多线程环境下共享一个 SimpleDateFormat 实例会产生并发问题。解决方案是使用 ThreadLocal 或者每次使用时都创建新实例。
2. 常用方法
方法签名 描述
SimpleDateFormat(String pattern) 使用给定的模式构造 SimpleDateFormat。
String format(Date date) 将 Date 对象格式化为字符串。

7.2 JDK 8及之后的新时间API

7.2.1 核心类概览

本地日期/时间:

  • LocalDate: 只表示日期，如 2024-04-17。
  • LocalTime: 只表示时间，如 15:45:30.123。
  • LocalDateTime: 表示日期和时间，是上面两者的结合，如 2024-04-17T15:45:30.123。它们都不包含时区信息。

带时区的日期/时间:

  • ZonedDateTime: 最完整的日期时间表示，包含时区信息。
  • ZoneId: 表示时区。

时间戳:

  • Instant: 表示时间线上的一个瞬时点（时间戳），以UTC为基准。与 Date 的 getTime() 类似，但精度更高（纳秒）。

时间段:

  • Duration: 表示以秒和纳秒为单位的时间间隔，适合机器计算（例如，两小时三十分钟）。
  • Period: 表示以年、月、日为单位的时间间隔，适合人类阅读（例如，两年三个月零五天）。

格式化与解析:

  • DateTimeFormatter: java.time 的格式化器，不可变且线程安全。

工具类:

  • TemporalAdjusters: 提供了一系列静态方法，用于进行复杂的日期调整，如“当月第一天”、“下个周日”等。
